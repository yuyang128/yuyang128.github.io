Concept 概念

This lesson presents the basics for understanding the transfer of data between the browser and the web application and how to trap a request/response with a HTTP proxy.

本课介绍了理解浏览器与网页应用之间数据传输的基础知识，以及如何用 HTTP 代理捕获请求/响应。

Goals 目标

The user should become familiar with the features of WebGoat by manipulating the above buttons to view hints, show the HTTP request parameters, the HTTP request cookies, and the Java source code. You can use the default Developer Tools in your browser to view the HTTP request and response.用户应通过作上述按钮，查看提示、显示 HTTP 请求参数、HTTP 请求 Cookie 和 Java 源代码，熟悉 WebGoat 的功能。你可以在浏览器中使用默认的开发者工具查看 HTTP 请求和响应。

How HTTP works: HTTP 的工作原理：

All HTTP transactions follow the same general format. Each client request and server response has three parts: the request or response line, a header section and the entity body.所有 HTTP 事务遵循相同的通用格式。每个客户端请求和服务器响应包含三个部分：请求或响应行、头部部分和实体体。

The client initiates a transaction as follows:客户端发起交易的方式如下：

  • The client contacts the server and sends a document request. A GET request can have url parameters and those parameters will be available in the web access logs.客户端联系服务器并发送文档请求。GET 请求可以有 URL 参数，这些参数会在网页访问日志中提供。
    • GET /index.html?param=value HTTP/1.0GET /index.html？param=value HTTP/1.0
  • Next, the client sends optional header information to inform the server of its configuration and the document formats it will accept.接着，客户端发送可选的头部信息，以告知服务器其配置及其可接受的文档格式。
    • User-Agent: Mozilla/4.06 Accept: image/gif,image/jpeg,用户代理：Mozilla/4.06 接受：image/gif，image/jpeg，**/**
  • In a POST request, the user supplied data will follow the optional headers and is not part of the contained within the POST URL.在 POST 请求中，用户提供的数据将跟随可选的头部，而不是 POST URL 中包含的内容。
